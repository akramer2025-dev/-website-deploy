## 🎯 كيفية استخدام القوالب

### للبيع للعملاء

1. **افتح المشروع مع العميل**
   - اعرض عليه نماذج من القوالب المتاحة
   - حدد القالب المناسب لاحتياجاته

2. **قم بالتنصيب**
   - نصّب القالب على استضافة العميل
   - فعّل جميع الإضافات المرفقة
   - استورد قالب جاهز قريب من احتياجاته

3. **خصّص الموقع**
   - غيّر الألوان والخطوط حسب هوية العميل
   - عدّل المحتوى (نصوص، صور، روابط)
   - أضف صفحات إضافية إذا لزم الأمر

4. **التسليم**
   - راجع الموقع على أجهزة مختلفة
   - سلّم الموقع مع دليل استخدام بسيط
   - قدم دعم فني للأسبوع الأول

### للاستخدام الشخصي

1. **اختر القالب المناسب**
   - حسب نوع مشروعك (متجر، مدونة، شركة، إلخ)

2. **نصّب وفعّل**
   - اتبع دليل التنصيب في ملف README.md الخاص بكل قالب

3. **خصّص حسب احتياجاتك**
   - استخدم أدوات التخصيص المدمجة
   - أضف المحتوى الخاص بك

## ⚠️ نصائح مهمة قبل البيع

### ✅ ما يجب فعله:
1. **تحقق من متطلبات الاستضافة** - تأكد أن استضافة العميل تدعم المتطلبات
2. **خذ نسخة احتياطية دائماً** - قبل أي تعديل كبير
3. **اختبر على أجهزة مختلفة** - موبايل، تابلت، ديسكتوب
4. **وضّح حدود الخدمة** - ما يشمله السعر وما لا يشمله
5. **قدم عقد بسيط** - يوضح الخدمات والأسعار والمسؤوليات

### ❌ ما يجب تجنبه:
1. **لا تبيع القالب كمنتج منفصل** - فقط المواقع المبنية عليه
2. **لا تعد بما لا تستطيع** - كن واقعياً في الجداول الزمنية
3. **لا تهمل الاختبار** - اختبر كل شيء قبل التسليم
4. **لا تنسى التوثيق** - اترك تعليمات واضحة للعميل
5. **لا تتجاهل الأمان** - فعّل SSL ونصّب إضافات الأمان

## 🔄 إضافة قوالب جديدة

عند شراء قوالب جديدة:

1. **أنشئ مجلد جديد**
```
templates/
├── xtra-theme/
├── new-theme-name/     ← المجلد الجديد
```

2. **انسخ الملفات**
   - Theme files
   - Plugins
   - Documentation

3. **أنشئ ملف README.md**
   - استخدم نموذج `xtra-theme/README.md` كمرجع
   - وثّق جميع المميزات والإضافات
   - اشرح خطوات التنصيب

4. **حدّث قاعدة البيانات في الموقع**
   - عدّل ملف `js/ready-websites.js`
   - أضف معلومات القالب الجديد في array `templatesData`
