# 📊 نظام الخطط التسويقية

دليل استخدام نظام إدارة الخطط التسويقية للشركات في لوحة التحكم.

---

## 🎯 نظرة عامة

نظام متكامل لإنشاء وإدارة خطط تسويقية احترافية للشركات مع إمكانية مشاركتها عبر رابط مخصص.

---

## ✨ المميزات

### 1. إدارة كاملة للخطط
- ✅ إنشاء خطط تسويقية مخصصة لكل شركة
- ✅ تعديل وحذف الخطط
- ✅ تتبع حالة كل خطة (مسودة، مكتملة، تم الإرسال)
- ✅ عرض جميع الخطط في CRM احترافي

### 2. معلومات شاملة
يحتوي كل خطة على:
- 🏢 **معلومات الشركة**: الاسم، البريد، الهاتف، المجال
- 🎯 **الهدف التسويقي**: هدف محدد وقابل للقياس
- 💰 **الميزانية والمدة**: تحديد الموارد والإطار الزمني
- 📱 **المنصات المستهدفة**: Facebook, Instagram, TikTok, Snapchat, Twitter, Google Ads
- 💡 **الاستراتيجيات**: خطة عمل تفصيلية
- 📊 **الأدوات والمقاييس**: أدوات التحليل ومؤشرات الأداء

### 3. عرض احترافي
- 🎨 تصميم احترافي جذاب للخطة
- 📄 إمكانية الطباعة أو الحفظ كـ PDF
- 📱 متوافق مع الموبايل
- 🔗 رابط مخصص لكل خطة

## 🚀 كيفية الاستخدام

### خطوة 1: تنفيذ الـ Schema في Supabase

1. افتح **Supabase Dashboard**
2. اذهب إلى **SQL Editor**
3. انسخ محتوى ملف `database/supabase-schema.sql` (القسم الخاص بـ marketing_plans)
4. نفّذ الكود

أو نفّذ هذا الكود مباشرة:

```sql
CREATE TABLE IF NOT EXISTS marketing_plans (
    id TEXT PRIMARY KEY,
    company_name TEXT NOT NULL,
    company_email TEXT NOT NULL,
    company_phone TEXT,
    company_industry TEXT,
    plan_data JSONB,
    status TEXT DEFAULT 'draft',
    view_link TEXT,
    created_at TIMESTAMP WITH TIME ZONE DEFAULT NOW(),
    updated_at TIMESTAMP WITH TIME ZONE DEFAULT NOW()
);

CREATE INDEX IF NOT EXISTS idx_marketing_plans_company ON marketing_plans(company_name);
CREATE INDEX IF NOT EXISTS idx_marketing_plans_status ON marketing_plans(status);
CREATE INDEX IF NOT EXISTS idx_marketing_plans_created ON marketing_plans(created_at);

ALTER TABLE marketing_plans ENABLE ROW LEVEL SECURITY;

CREATE POLICY "Enable all access for marketing_plans"
    ON marketing_plans FOR ALL
    USING (true) WITH CHECK (true);

CREATE TRIGGER update_marketing_plans_updated_at
    BEFORE UPDATE ON marketing_plans
    FOR EACH ROW
    EXECUTE FUNCTION update_updated_at_column();
```

### خطوة 2: الوصول للنظام

1. افتح لوحة التحكم (admin/index.html)
2. من القائمة الجانبية، اضغط على **"الخطط التسويقية"**
3. ستظهر لك شبكة CRM بجميع الخطط

### خطوة 3: إنشاء خطة جديدة

#### من قسم الخطط التسويقية:
1. اضغط على زر **"خطة تسويقية جديدة"** (الزر الأخضر)

#### من الإجراءات السريعة (Dashboard):
1. اضغط على **"خطة تسويقية جديدة"** في قسم الإجراءات السريعة

#### ملء البيانات:

**📋 معلومات الشركة:**
- اسم الشركة (إجباري)
- البريد الإلكتروني (إجباري)
- رقم الهاتف (اختياري)
- المجال (إجباري): اختر من القائمة (تجارة إلكترونية، مطاعم، عقارات، إلخ)

**🎯 تفاصيل الخطة:**
- الهدف التسويقي (إجباري): مثال "زيادة المبيعات بنسبة 50% خلال 3 أشهر"
- الميزانية الشهرية (اختياري): بالجنيه
- المدة (اختياري): عدد الشهور
- المنصات المستهدفة: اختر واحدة أو أكثر
- الاستراتيجيات والتوصيات: خطة العمل التفصيلية
- الأدوات والمقاييس: أدوات التحليل و KPIs
- ملاحظات إضافية (اختياري)

2. اضغط **"حفظ الخطة"**

### خطوة 4: إدارة الخطط

في شبكة CRM، كل بطاقة خطة تحتوي على:

**📊 معلومات سريعة:**
- اسم الشركة
- المجال
- البريد الإلكتروني
- الهاتف
- تاريخ الإنشاء
- حالة الخطة (مسودة، مكتملة، تم الإرسال)
- أيقونات المنصات المستهدفة

**⚡ أزرار الإجراءات:**
- **تعديل**: لتحديث بيانات الخطة
- **عرض**: لفتح صفحة العرض الاحترافية (يظهر للخطط المكتملة فقط)
- **إرسال**: لإرسال الخطة للعميل عبر البريد الإلكتروني (يظهر للخطط المكتملة فقط)
- **حذف**: لحذف الخطة نهائياً

### خطوة 5: مشاركة الخطة مع العميل

#### الطريقة 1: الإرسال التلقائي
1. اضغط على زر **"إرسال"** في بطاقة الخطة
2. سيفتح تطبيق البريد تلقائياً مع:
   - عنوان مكتوب مسبقاً
   - رسالة احترافية
   - رابط الخطة
3. أرسل البريد

#### الطريقة 2: نسخ الرابط
1. اضغط على **"عرض"** لفتح صفحة الخطة
2. انسخ الرابط من المتصفح
3. شارك الرابط مع العميل عبر أي وسيلة (واتساب، بريد، إلخ)

## 🎯 حالات الخطة

النظام يدعم 3 حالات للخطة:

### 1. مسودة (draft)
- خطة قيد الإنشاء
- يظهر badge أصفر
- متاح: تعديل، حذف

### 2. مكتملة (completed)
- خطة جاهزة للإرسال
- يظهر badge أخضر
- متاح: تعديل، عرض، إرسال، حذف

### 3. تم الإرسال (sent)
- خطة تم إرسالها للعميل
- يظهر badge أزرق
- متاح: تعديل، عرض، حذف

## 🔧 Troubleshooting

### مشكلة: الخطط لا تظهر
**الحل:**
1. تأكد من تنفيذ الـ schema في Supabase
2. افتح Console (F12) وتحقق من الأخطاء
3. تأكد من صحة اتصال Supabase

### مشكلة: صفحة العرض فارغة
**الحل:**
1. تأكد من وجود `id` في الرابط: `?id=plan_xxxxx`
2. تحقق من صحة الـ plan_id في قاعدة البيانات
3. تأكد من تكوين `js/supabase-config.js` صحيح

### مشكلة: الإرسال لا يعمل
**الحل:**
1. تأكد من وجود `view_link` في قاعدة البيانات
2. تحقق من صحة البريد الإلكتروني للشركة

## 📊 بنية الجدول في Supabase

```sql
marketing_plans
├── id (TEXT, PRIMARY KEY)          # plan_1234567890
├── company_name (TEXT, NOT NULL)   # اسم الشركة
├── company_email (TEXT, NOT NULL)  # البريد الإلكتروني
├── company_phone (TEXT)            # رقم الهاتف
├── company_industry (TEXT)         # المجال
├── plan_data (JSONB)               # بيانات الخطة كـ JSON
│   ├── goal                        # الهدف التسويقي
│   ├── budget                      # الميزانية
│   ├── duration                    # المدة
│   ├── platforms                   # المنصات (array)
│   ├── strategy                    # الاستراتيجيات
│   ├── metrics                     # الأدوات والمقاييس
│   └── notes                       # ملاحظات
├── status (TEXT)                   # draft, completed, sent
├── view_link (TEXT)                # رابط العرض
├── created_at (TIMESTAMP)          # تاريخ الإنشاء
└── updated_at (TIMESTAMP)          # تاريخ آخر تحديث
```
