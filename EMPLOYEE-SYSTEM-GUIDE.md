# 📋 دليل نظام إدارة الموظفين

## 🎯 نظرة عامة

تم إنشاء نظام كامل لإدارة الموظفين يشمل:
- ✅ إضافة موظفين جدد من لوحة الأدمن
- ✅ إرسال رابط تسجيل للموظف عبر WhatsApp أو Email
- ✅ صفحة تسجيل بيانات الموظف الكاملة
- ✅ رفع المستندات (البطاقة - السيلفي)
- ✅ التوقيع الإلكتروني على العقد
- ✅ مراجعة واعتماد طلبات الموظفين

---

## 📁 الملفات الجديدة

### 1. **employee-onboarding.html**
صفحة تسجيل بيانات الموظف - تحتوي على:
- نموذج من 4 خطوات (Multi-Step Form)
- البيانات الشخصية
- رفع صورة البطاقة (وجهين)
- التقاط صورة سيلفي من الكاميرا
- عرض العقد والتوقيع الإلكتروني

### 2. **admin/index.html (تم التعديل)**
إضافة نظام طلبات الموظفين:
- قسم جديد "طلبات الموظفين" في السايدبار
- صفحة عرض جميع الطلبات
- أزرار قبول/رفض الطلبات
- عرض تفاصيل كل طلب مع الصور والمستندات

---

## 🚀 كيفية الاستخدام

### الخطوة 1️⃣: إضافة موظف جديد (من الأدمن)

1. ادخل على لوحة الإدارة: `admin/index.html`
2. اضغط على "الموظفون" من القائمة الجانبية
3. اضغط على زر **"إضافة موظف جديد"**
4. أدخل البيانات المطلوبة:
   - البريد الإلكتروني
   - اسم الموظف
   - الراتب الشهري

5. سيظهر لك رابط التسجيل - **اختر أحد الخيارات:**
   - ✅ نسخ الرابط وإرساله عبر Email
   - ✅ إرساله فوراً عبر WhatsApp (سيطلب رقم الهاتف)

**مثال على الرابط:**
```
https://yoursite.com/employee-onboarding.html?id=emp_1234567890
```

---

### الخطوة 2️⃣: الموظف يكمل بياناته

عندما يفتح الموظف الرابط، سيجد صفحة من 4 خطوات:

#### **الخطوة الأولى: البيانات الشخصية** 📝
- الاسم الكامل
- رقم الهاتف
- البريد الإلكتروني (معبأ مسبقاً)
- العنوان
- رقم البطاقة (14 رقم)
- المسمى الوظيفي

#### **الخطوة الثانية: رفع صورة البطاقة** 🆔
- صورة البطاقة الأمامية
- صورة البطاقة الخلفية
- معاينة الصور قبل الرفع

#### **الخطوة الثالثة: الصورة الشخصية** 📸
- فتح الكاميرا
- التقاط صورة سيلفي
- إعادة التصوير إذا لزم الأمر

#### **الخطوة الرابعة: العقد والتوقيع** ✍️
- قراءة بنود العقد
- العقد يحتوي على:
  - اسم الموظف
  - المسمى الوظيفي
  - الراتب الشهري
  - مدة العقد (شهر تجريبي)
  - تاريخ البدء والانتهاء
- الموافقة على الشروط
- التوقيع الإلكتروني على Canvas

بعد إرسال الطلب، تظهر رسالة نجاح ✅

---

### الخطوة 3️⃣: مراجعة الطلبات (من الأدمن)

1. ادخل على لوحة الإدارة
2. اضغط على **"طلبات الموظفين"** من القائمة الجانبية
3. ستجد قائمة بجميع الطلبات:
   - ⏳ **قيد المراجعة** (لم تتم الموافقة بعد)
   - ✅ **تم القبول**
   - ❌ **تم الرفض**

4. لكل طلب، يمكنك:
   - عرض كل التفاصيل
   - مشاهدة الصور والمستندات
   - قبول الطلب ← سيُضاف الموظف تلقائياً لقائمة الموظفين
   - رفض الطلب (مع سبب اختياري)

5. **إذا لم يكمل الموظف بياناته:**
   - اضغط على **"إعادة إرسال الرابط"**
   - أدخل رقم WhatsApp
   - سيُرسل تذكير تلقائي

## 🔥 المميزات

### ✨ تصميم احترافي
- **Multi-Step Progress Bar** - شريط تقدم تفاعلي
- **Drag & Drop Upload** - رفع الصور بالسحب والإفلات
- **Live Camera** - التقاط صورة من الكاميرا مباشرة
- **Digital Signature** - توقيع إلكتروني على Canvas
- **Responsive** - متجاوب مع الهواتف والأجهزة اللوحية

### 🔒 الأمان
- البيانات مشفرة في Firebase Firestore
- الصور محفوظة في Firebase Storage
- لكل موظف ID فريد
- التحقق من صحة البيانات قبل الإرسال

### 📊 التقارير
- عدد الطلبات المعلقة (في الـ Badge)
- تاريخ كل طلب
- حالة كل موظف (قيد المراجعة / تم القبول / تم الرفض)

## 📝 بنود العقد

العقد يتم توليده تلقائياً ويحتوي على:

1. **اسم الموظف** - من البيانات المدخلة
2. **تاريخ البدء** - تلقائي (اليوم)
3. **تاريخ الانتهاء** - بعد شهر واحد
4. **الراتب** - من البيانات التي أدخلها الأدمن
5. **المسمى الوظيفي** - من بيانات الموظف
6. **الشروط والأحكام**:
   - فترة تجريبية شهر واحد
   - ساعات العمل: 9 صباحاً - 5 مساءً
   - أيام العمل: الأحد - الخميس
   - يحق لأي من الطرفين إنهاء العقد في الفترة التجريبية
   - الراتب يُصرف في نهاية الشهر بعد تقييم الأداء

## 🚨 ملاحظات هامة

1. **يجب أن يكون Firebase مُعد بشكل صحيح** في:
   - `admin/index.html`
   - `employee-onboarding.html`

2. **Firestore Rules** - تأكد من أن القواعد تسمح بالقراءة والكتابة:
```javascript
rules_version = '2';
service cloud.firestore {
  match /databases/{database}/documents {
    match /employees/{empId} {
      allow read, write: if true; // للتطوير فقط - عدّل القواعد في الإنتاج
    }
  }
}
```

3. **Storage Rules** - تأكد من السماح برفع الصور:
```javascript
rules_version = '2';
service firebase.storage {
  match /b/{bucket}/o {
    match /employees/{empId}/{allPaths=**} {
      allow read, write: if true; // للتطوير فقط
    }
  }
}
```
